Gesture-Driven Robotic Arm.

Title: Gesture-Driven Robotic Arm: Utilizing Computer Vision for Enhanced Human-Machine Interaction.
Creators: Md. J. Raihan
Abstract

This project presents a computer vision-controlled robot arm system. It consists of a computer that continuously captures real-time camera input and employs a MediaPipe Machine Learning model to recognize hand gestures. The recognized gestures are then transmitted to an ESP-32 microcontroller via the WebSocket communication protocol. The robot arm is equipped with three stepper motors, responsible for controlling the base, Arm 1, and Arm 2. A 12V, 5A power supply is used to provide the necessary power to the stepper motors, while a buck converter steps down the voltage to 3.3V for the ESP-32.

Hand gestures play a crucial role in controlling the robot arm. A rectangular box, adaptable to any location within the camera's view, activates the control mechanism when a single hand is displayed. Specific gestures control individual robot arm components, with fist or open palm gestures used for vertical arm control and left/right placement determining base rotation. The system supports simultaneous control of multiple components, enhancing its efficiency and versatility.

The robot arm's physical design is crafted using PVC board, nuts, bolts, and glue, with the base featuring a 360-degree rotation capability. The stepper motors and robot arm are connected via couplers. Both the microcontroller (ESP-32) and the computer require custom coding: C++ is used for the ESP-32, utilizing the Accelstepper library for motor control, and network libraries to enable WebSocket communication. An HTML webpage is created to facilitate remote robot arm control.

The computer vision aspect of the project employs Python to process camera input. A Mediapipe human pose prediction model identifies the hand's joint coordinates relative to the screen. A rectangular box activation control ensures that gestures are recognized only when the hand is outside the box. A four-finger gesture is used to disable the control. This user-friendly gesture control mechanism makes the robot arm versatile and accessible for various applications.

Automated Hand Exercise Machine.

Title: Automated Hand Exercise Machine with Monitoring System for Paralysis Patients.
Creators: Md. J. Raihan, Md. Irfan Hussain Sikder, Swaraj Mondal
Abstract

We have divided the whole project into three main parts, those are hardware, circuit diagram and software parts. In the hardware parts we have setup the motor with the human body. To do that we have designed a motor shaft to attach a lather from the wrist to that motor shaft. Next to keep the motor attach into a block box we have designed a motor mount that attaches the motor to the box. Also, to keep the box attach to the human shoulder we designed an outfit using lather that attaches the box to the shoulder as shown in the Figure. We have used a rubber finger gripper to keep the heart rate sensor attach to the fingertip. In the circuit part, we have designed a circuit diagram to read the value from the heart rate sensor and to control the motor. An Arduino has been used as the microcontroller to connect all the electrical component. The motor driver has been used to control the motor direction and speed. A Bluetooth device has been connected to the microcontroller to read and send data from and to the computer. In the software part, we send the heart rate sensor data to the computer via Bluetooth and receive the motor control input from the computer. To make the hand exercise automatic we had to detect weather the hand is in down position or in up position. To achieve that we have implemented a pose detection algorithm using the MediaPipe library from google. The pose detection algorithm gives us the human body joint land mark data using which we have extracted the shoulder, elbow and wrist land mark value. Further we have used these values to calculate angle between the biceps and forearm and using the angle we determine whether the hand is in down position or up position. If the hand is in down position a command is send to the microcontroller to use the motor so that the motor pick ups the hand. As the hand is in up position another command is seed to the microcontroller to cut the power to the motor. At this point the patient can apply force to bring the hand down to lower position. This process continues and the patient gets an automatic hand exercise. Further we have built interface to count the number hand exercise and show them in real-time in a window along with the patientâ€™s heart rate data. This shows the health condition of the patient as he continuous to do the exercise.
